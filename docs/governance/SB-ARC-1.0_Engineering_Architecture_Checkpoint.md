\# Repository Strategy

GitHub remains the canonical implementation repository.

The repository shall contain:

\- application source code,  
\- governance documentation,  
\- implementation documentation,  
\- historical records,  
\- engineering reference material.

Implementation shall remain repository-first.

Whenever practical, AI systems shall consume repository context rather than duplicate documentation.

\# Engineering Principles

The following principles are reaffirmed.

\#\# Repository First

The connected GitHub repository is the canonical implementation environment.

\---

\#\# Evidence Over Assumptions

Engineering decisions shall be based upon:

\- runtime evidence,  
\- repository evidence,  
\- platform evidence,  
\- documented observations.

Verification reports shall not be produced without supporting evidence.

\---

\#\# Human-in-the-Loop

Humans retain ownership of:

\- Product Truth,  
\- governance,  
\- implementation authorization,  
\- business decisions.

AI supports implementation.

AI does not replace governance.

\---

\#\# Incremental Governed Delivery

Implementation shall proceed through independently reviewable milestones.

Each implementation phase shall conclude before the next begins.

\---

\#\# Separation of Responsibilities

Governance and implementation shall remain independent.

Mission Control governs.

Claude Engineering advises.

Lovable builds.

Supabase provides backend services.

No platform shall assume responsibilities assigned to another.

\---

\# Controlled Evolution

Changes to:

\- governance,  
\- engineering workflow,  
\- repository organization,  
\- AI platform responsibilities,  
\- implementation methodology,

shall occur only through Mission Control–authorized architectural review.

Routine implementation missions shall not redefine the engineering operating model.
